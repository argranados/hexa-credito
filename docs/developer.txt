- Clean Architecture and Hexagonal Architecture 
share the exact same core goal: keeping your business logic decoupled from frameworks, databases, and user interfaces

- Hexagonal focuses on how your application connects to external components. 
Clean Architecture focuses on how your application enforces structural boundaries throughout the entire codebase

- Heagonal. to keep your core logic separated from database and API frameworks
Clean Architecture, want rigid, standardized rules governing exactly where use-case logic, entities, and controllers live

- cada puerto de entrada representa una intención de negocio, no una operación CRUD genérica.
